полноценный игровой движок. Каждый этап логически вытекает из предыдущего, максимально используя уже написанную математику BSP и BVH.
1. CSG & Прототипирование (Геометрия)
Цель: Перейти от «накладывания» объектов к их «вычитанию» (создание проемов, арок).

    Логика вычитания (Subtraction):
        Реализуй тип браша BrushType::Subtractive (Brush-carver).
        В функции build() при пересечении двух брашей используй плоскости «вычитающего» браша для разделения граней «основного» браша.
        Грани, оказавшиеся внутри объема вычитающего браша, должны отбрасываться при генерации меша.
    Группировка объектов: Создание пресетов (Prefab), состоящих из нескольких брашей (например, готовый дверной проем).
    Grid Rendering: Шейдерная сетка на бесконечной плоскости
    для точного позиционирования вычитающих объемов.

2. Entity System & Триггеры (Логика)
Цель: Оживить мир через интерактивные объекты.

    Базовый класс Entity:
        Создай структуру для объектов, не являющихся геометрией (точки спавна, источники света, предметы).
        Интеграция с BVH для выделения энтити кликом мыши в редакторе.
    Система триггеров (Trigger Brushes):
        Браши с флагом is_ghost = true (проходимые), но вызывающие событие on_enter(entity) при срабатывании коллизии.
        Реализация простых скриптов: телепорт (перенос координат), лифт (изменение pos браша со временем), зона урона.

3. Звуковой движок (OpenAL / SDL_mixer)
Цель: Добавить 3D-аудио и звуковые эффекты импакта.

    Выбор API:
        SDL_mixer: Проще для 2D-звуков и музыки (подходит для простых эффектов).
        OpenAL: Лучше для 3D-позиционирования (звук за стеной, затухание от расстояния).
    Реализация:
        Интеграция с частицами: При вызове spawn_spark_burst запускай случайный звук из пула metal_hit_01.wav.
        Шаги: Проверка is_moving && is_grounded для воспроизведения звука ходьбы.
        Фоновый эмбиент: Цикличный звук ветра или гула ламп в хабе.

4. Освещение и запекание (Графика)
Цель: Оптимизация и создание атмосферы.

    Vertex Lighting:
        В момент build() для каждой вершины вычисляй lightAccum от всех ламп.
        Сохраняй результат в атрибут aVertexColor (больше не нужно считать свет в фрагментном шейдере для статики).
    Static Shadow Maps / Lightmaps:
        Используй рейкаст через BSP для проверки теней: если луч от вершины до лампы пересекает Solid-лист — вершина в тени.

5. Ресурсная система (ZIP)
Цель: Уход от захардкоженных путей и организация данных.

    Virtual File System (VFS):
        Библиотека zip или tar в Rust для чтения архивов без распаковки на диск.
        Парсинг файлов текстур и шейдеров из одного архива assets.zip.
    Автоматизация: Загрузка всех .txt файлов из папки maps/ в меню выбора уровня.

Как двигаться дальше?
Рекомендую начать со Звукового движка (3.1). Это самый быстрый способ получить «вау-эффект» — когда твои искры начнут звучать, иммерсивность подскочит мгновенно.
